Engineering Highlights

The decisions that make this a portfolio piece.

The kind of details that separate a notebook from a system you'd run in production.

⏳

Look-ahead bias guard

An empirical pytest fixture perturbs future data and asserts day-t features don't change. Catches the most expensive bug in quant pipelines automatically.

♻︎

Content-hashed idempotency

Re-running ingestion on the same date range writes zero new bytes. Partitions are sorted, serialized, SHA-256'd, and only replaced on real diff.

📊

Walk-forward CV, never random splits

LightGBM trained with TimeSeriesSplit, expanding window, early stopping on the validation fold. Per-fold macro-F1 logged to MLflow.

🧮

BIC-selected HMM

Fit Gaussian HMMs at K∈{3,4} and pick by Bayesian Information Criterion — automatic model-complexity selection, not a magic number.

💸

Realistic cost model

Base + size-impact slippage (linear in order/ADV), configurable borrow drag on short legs, all vectorised for fast backtests.

📈

Bootstrap Sharpe CIs

1000-resample 95% confidence intervals on every OOS Sharpe number. No more "0.42 looks good" — quantify the uncertainty.

🔒

Strict typing, ≥80% coverage

mypy --strict across all of src/, Pydantic v2 at every external boundary, coverage gate enforced in CI.

📡

Drift monitoring

Weekly Prefect flow scores last week's predictions vs realized regime and fires an ntfy.sh webhook (free, no account) if F1 drops >10pp.

🐳

One-command bring-up

make setup && make up from a clean clone spins up Streamlit + MLflow + Prefect via docker-compose with healthchecks.

A representative snippet

The look-ahead bias guard.

An empirical test, not a comment. If anyone introduces a feature that uses future data, this fails loudly in CI.

tests/unit/test_look_ahead_bias.py
def test_no_feature_column_can_be_computed_from_future_data(
    synthetic_mart: pl.DataFrame,
) -> None:
    """Empirical sampler: perturb the *future* of the source mart and re-build
    features. None of the lagged feature values for the chosen date may change."""
    target_date = date(2024, 1, 10)
    perturbed = synthetic_mart.with_columns(
        pl.when(pl.col("trade_date") > target_date)
        .then(pl.col("adj_close") * 1.5)
        .otherwise(pl.col("adj_close"))
        .alias("adj_close")
    )

    orig = build_supervised_features(synthetic_mart).filter(
        pl.col("trade_date") == target_date
    )
    new  = build_supervised_features(perturbed).filter(
        pl.col("trade_date") == target_date
    )

    for col in orig.columns:
        if col in {"ticker", "trade_date"}:
            continue
        assert orig[col].to_list() == new[col].to_list(), (
            f"feature `{col}` changed after future-only perturbation"
        )
